## 阿慶的写作风格特征（持续学习更新）

### 硬性约束条件
- **字数控制：800字以内**（2026-02-14更新）
- **排版要求：文字图片化**，每图约80字（即7-8张图）

### 当前已观察到的特征
- **重视真实经历** > 抽象理论
- **语言口语化、有温度**（使用语气词、短句）
- **第一人称"我"** 增强个人色彩
- **与读者直接对话**（增加提问互动）
- **金句提炼**（核心观点浓缩成一句话）

### 第2期复盘新增特征（2026-02-11）

**叙事结构偏好：**
- **时间线叙事**：所有事件标注明确年份（2014、2016、2018、2020）
- **三次转折结构**：三次失败/尝试，前两次循环，第三次觉醒
- **具体事件 → 内心转变 → 行动改变** 的递进

**语言细节特征：**
- **善用括号补充内心OS**：如"（这其实又走回了老路...）"、"（最后被人卖了还在帮人数钞票）"
- **数字具体化**：不说"很久"说"2个月"，不说"很努力"说"每天14小时"
- **感叹号和短句**：增强口语感和讲故事的氛围
- **具体引用**：引用具体书名和人名（星云大师《成就的秘密》、罗永浩、比尔·盖茨《创新之路》）

**情感表达特征：**
- **不避讳脆弱**："时隔多年依然会有泪目之感"
- **场景化描写**："如留守儿童一般，睡在自己的办公室里"
- **自嘲式幽默**：承认当年的天真和错误

## 子代理优化方案（2026-02-13）

基于 Kimi K2.5 特性，为三个子代理制定的具体优化策略。

### 公众号助手（Kimi K2.5）

**长文生成技巧：**
- 分段生成：大纲 → 每段内容 → 润色 → 合并
- 避免一次性生成整篇文章

**风格一致性：**
- 把阿慶写作DNA整理成 JSON 格式给助手
- 结构化数据比口述更稳定

**减少幻觉：**
- 强制要求每篇文章必须有时间锚点和具体数字
- 禁止模糊的"很久以前"、"很多人"

### 修仙开发助手（Kimi K2.5）

**上下文管理：**
- 每次只处理1个API，完成后再下一个
- 不要一次让写完整模块

**代码质量提升：**
- 三步法：先写接口文档 → 再写实现 → 最后写单元测试
- 强制使用 TypeScript 严格模式
- 所有函数必须有明确的返回类型

**类型安全：**
- 提供 Prisma schema 作为参考
- 要求生成对应的 TypeScript 类型定义

### 其他助手（Kimi K2.5）

**实验性任务：**
- 小步快跑：最小版本 → 测试 → 迭代
- 不要追求完美，先跑通再优化

**工具开发规范：**
- 必须提供使用示例
- 必须提供测试用例
- 代码要有清晰的注释

**详细文档：** `memory/2026-02-13-助手优化方案.md`

### 日历使用建议
- **plan日历**：每周日晚上规划下周
- **work日历**：每天记录实际工作内容
- **welldone日历**：每晚完成一件事后记录，获得成就感
- **improving日历**：每周复盘，只记录一个改进点
- **debt日历**：每月更新一次，看到减少趋势
- **dao日历**：每天修行时间记录，哪怕只有10分钟
