# 模型用量监控方案

> 创建时间：2026年2月8日  
> 监控模型：OpenClaw官方通道 + Moonshot官方API

---

## 📊 监控概览

| 模型 | 监控方式 | 自动警报 | 备注 |
|------|----------|----------|------|
| **OpenClaw官方通道** (kimi-coding/k2p5) | ✅ 运行状态 | ⚠️ 用量需查控制台 | 由OpenClaw平台管理 |
| **Moonshot官方API** (moonshot/kimi-k2-5) | ✅ 实时余额 | ✅ 低余额警报 | 每小时自动检查 |

---

## 🔔 警报机制

### 监控脚本位置
```
~/.openclaw/workspace/scripts/monitor_models.sh
```

### 检查频率
- **余额检查**：每小时一次
- **日报生成**：每天上午8点

### 警报阈值
- **Moonshot余额**：低于 ¥10 时发送警报

### 警报接收方式
1. **飞书消息**：虾哥会主动发送警报消息
2. **警报文件**：保存在 `~/.openclaw/workspace/alerts/`
3. **历史记录**：保存在 `~/.openclaw/workspace/logs/`

---

## 📁 文件结构

```
~/.openclaw/workspace/
├── scripts/
│   └── monitor_models.sh          # 主监控脚本
├── alerts/
│   └── moonshot_alert.txt         # 余额不足警报（如有）
└── logs/
    ├── moonshot_balance_2026-02-08.csv  # 每日余额记录
    └── daily_report_20260208.txt        # 每日报告
```

---

## 🔧 手动操作

### 立即检查余额
```bash
~/.openclaw/workspace/scripts/monitor_models.sh
```

### 查看当前警报
```bash
ls -la ~/.openclaw/workspace/alerts/
```

### 查看余额历史
```bash
cat ~/.openclaw/workspace/logs/moonshot_balance_$(date +%Y-%m-%d).csv
```

## ⚠️ 注意事项

### OpenClaw官方通道
- ✅ **优点**：开箱即用，稳定性好
- ⚠️ **限制**：用量详情需登录 OpenClaw 控制台查看
- 💡 **建议**：适合日常使用，无需担心余额

### Moonshot官方API
- ✅ **优点**：自主可控，官方最新功能
- ⚠️ **注意**：需要自行充值管理余额
- 💰 **计费**：输入 ¥0.002/1K tokens，输出 ¥0.008/1K tokens
- 🔗 **充值地址**：https://platform.moonshot.cn

## 📈 用量估算

以每天使用100次对话为例：

| 模型 | 平均tokens/次 | 每日成本 |
|------|--------------|----------|
| OpenClaw通道 | ~2K | ¥0 (免费额度内) |
| Moonshot API | ~2K | ~¥0.2-0.5 |

**建议**：日常使用以OpenClaw通道为主，Moonshot作为备用。

---

## 🔄 恢复系统后配置

如果系统重装，需要：
1. 恢复 `~/.openclaw/workspace/` 目录
2. 确认脚本可执行：`chmod +x ~/.openclaw/workspace/scripts/*.sh`
3. 重新添加cron任务（已包含在系统备份文档中）
