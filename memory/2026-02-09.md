## Feishu 消息显示问题诊断

**问题**：飞书消息在网页端 WebChat 中不显示，但之前可以"切换视角"查看。

**诊断结果**：
- 飞书消息确实被正确记录在会话历史文件中
- 消息以 "System:" 前缀作为系统事件注入到主会话 (`agent:main:main`)
- 示例记录：
  - `System: [2026-02-09 04:59:55 GMT+8] Feishu DM from ou_af6154cf494e961b3199f31d3120188b: 怎么样？你改了吗？`
  - `System: [2026-02-09 07:42:49 GMT+8] Feishu DM from ou_af6154cf494e961b3199f31d3120188b: 早上好，蝦哥`

**根本原因**：
- Feishu 插件将所有消息作为系统事件注入主会话，而非创建独立的飞书会话
- WebChat UI 可能过滤或不显示系统事件消息
- 这是插件设计行为，不是配置错误

**解决方案建议**：
1. 使用 TUI (`openclaw tui`) 查看，支持显示 system notices
2. 检查 WebChat 是否有"显示系统消息"的选项
3. 如需在 WebChat 中查看，可能需要向插件开发者反馈
