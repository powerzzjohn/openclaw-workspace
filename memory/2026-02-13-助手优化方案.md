# 2026-02-13 - 助手优化方案

## 背景
今天根据 Kimi K2.5 的特性，为三个子代理制定了具体优化方案。

---

## 📝 公众号助手优化方案（Kimi K2.5）

### 长文生成技巧
- **分段生成**：大纲 → 每段内容 → 润色 → 合并
- 避免一次性生成整篇文章，防止超出上下文

### 风格一致性
- 把 MEMORY.md 里的阿慶写作DNA整理成 **JSON 格式** 给助手
- 结构化数据比口述更稳定

### 减少幻觉
- 强制要求每篇文章必须有 **时间锚点** 和 **具体数字**
- 禁止模糊的"很久以前"、"很多人"

---

## 🎮 修仙开发助手优化方案（Kimi K2.5）

### 上下文管理
- 每次只处理 **1个API**，完成后再下一个
- 不要一次让写完整模块

### 代码质量提升
- **三步法**：先写接口文档 → 再写实现 → 最后写单元测试
- 强制使用 TypeScript 严格模式
- 所有函数必须有明确的返回类型

### 类型安全
- 提供 Prisma schema 作为参考
- 要求生成对应的 TypeScript 类型定义

---

## 🔧 其他助手优化方案（Kimi K2.5）

### 实验性任务
- **小步快跑**：最小版本 → 测试 → 迭代
- 不要追求完美，先跑通再优化

### 工具开发规范
- 必须提供 **使用示例**
- 必须提供 **测试用例**
- 代码要有清晰的注释
