## 🤖 AI 模型配置（双模型详解）

我们配置了**两个Kimi K2.5模型**，互为备份，可根据需要切换使用。

---

### 模型一：OpenClaw 官方通道（当前默认）

| 配置项 | 详情 |
|--------|------|
| **模型ID** | `kimi-coding/k2p5` |
| **别名** | Kimi K2.5 |
| **提供商** | kimi-coding（OpenClaw官方集成） |
| **认证方式** | OpenClaw内置API Key |
| **上下文窗口** | 262,144 tokens（约20万字） |
| **最大输出** | 8,192 tokens |
| **输入支持** | 文本、图片 |
| **推理能力** | ✅ 支持深度思考 |
| **当前状态** | 🟢 主用模型 |

**特点：**
- ✅ 开箱即用，无需自己申请API
- ✅ 通常有更好的稳定性和配额
- ✅ 由OpenClaw团队维护
- ⚠️ 依赖OpenClaw服务可用性

---

| 配置项 | 详情 |
|--------|------|
| **模型ID** | `moonshot/kimi-k2-5` |
| **别名** | Kimi K2.5 (我的API) |
| **提供商** | Moonshot（月之暗面官方） |
| **API地址** | `https://api.moonshot.cn/v1` |
| **API Key** | `sk-...GYW52H` ⚠️ **需保密** |
| **上下文窗口** | 256,000 tokens |
| **最大输出** | 8,192 tokens |
| **输入支持** | 文本、图片 |
| **计费标准** | 输入 ¥0.002/1K tokens，输出 ¥0.008/1K tokens |
| **当前状态** | 🟡 备用模型 |

**特点：**
- ✅ 自主可控，不依赖第三方服务
- ✅ 可查看详细用量和账单
- ✅ 官方最新功能和模型优先体验
- ⚠️ 需要自行充值和管理API额度

### 两个模型的对比

| 对比项 | OpenClaw通道 | Moonshot官方API |
|--------|--------------|-----------------|
| **配置复杂度** | 简单（内置） | 中等（需申请API） |
| **成本控制** | 由OpenClaw管理 | 自行管理充值 |
| **稳定性** | 依赖OpenClaw | 依赖Moonshot官方 |
| **隐私性** | 经OpenClaw中转 | 直连官方 |
| **技术支持** | OpenClaw社区 | Moonshot官方 |

---

### 如何切换模型

**临时切换（单次对话）：**
```
使用模型 moonshot/kimi-k2-5
```

**查看当前模型：**
```
/status
```

**修改默认模型（需修改配置文件）：**
编辑 `~/.openclaw/openclaw.json` 中的 `agents.defaults.model.primary` 字段

## 🔄 系统恢复步骤

如果系统出现问题，按以下步骤恢复：

### 步骤1：重新安装 OpenClaw
```bash
npm install -g openclaw
```

### 步骤2：恢复配置文件
将备份的 `openclaw.json` 复制到：
```bash
~/.openclaw/openclaw.json
```

### 步骤3：恢复工作区文件
将备份的 workspace 文件夹复制到：
```bash
~/.openclaw/workspace/
```

### 步骤4：启动服务
```bash
openclaw gateway start
```

### 步骤5：验证
1. 发送飞书消息测试连接
2. 检查虾哥是否能读取历史记忆
